# Product

## Register

product

## Users

Solo developers using AI agents for code exploration, generation, and structured development workflows. They work in focused sessions — often deep in a codebase, switching between reading code, issuing commands, and reviewing generated artifacts. Context is everything; interruptions are costly.

## Product Purpose

Hank is a desktop AI agent environment for developers who want structured, traceable AI-assisted coding. It connects to multiple LLM providers, executes shell commands and file operations, and tracks changes through specs, checkpoints, and conversation branches. Success looks like: a developer finishes a complex multi-step coding task faster and with more confidence than working alone, with a clear audit trail of what happened.

## Brand Personality

Capable, calm, precise. An expert tool that stays out of the way. Confidence through restraint — never showy, never uncertain. The interface should feel like a well-tuned instrument: responsive, predictable, and invisible when things are going well.

## Anti-references

- Generic chatbot UIs: no speech bubbles, no avatars, no typing indicators, no "AI is thinking" spinners with personality. Hank is not a chat companion — it's a development environment.
- Rounded, playful SaaS aesthetics. No gratuitous gradients, no bouncy animations, no emoji-heavy copy.
- Electron apps that feel sluggish or web-first. Should feel native and immediate.

## Design Principles

1. **Density over decoration.** Show more information in less space. Every pixel should earn its place through utility, not ornament.
2. **Terminal confidence.** The interface should feel as decisive as a CLI — immediate feedback, no ambiguity about state, no unnecessary confirmation dialogs.
3. **Traceable by default.** Every action has a visible trail. Branching, checkpoints, and history are first-class, not hidden behind menus.
4. **Calm under load.** When the agent is doing complex work (multiple tool calls, long outputs), the UI stays composed. No visual panic.
5. **Keyboard-native.** Primary workflows should be achievable without a mouse. The interface rewards muscle memory.

## Accessibility & Inclusion

WCAG AA compliance as baseline. Additionally: full keyboard navigation for all primary workflows, reduced motion support (prefers-reduced-motion respected), sufficient contrast ratios in the dark theme, and high-contrast mode consideration for extended coding sessions.
