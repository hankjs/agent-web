## What IS observable (harness-injected context)

Although the base prompt is hidden, the transcript reveals how the harness
augments each conversation via `attachment` records injected after the first
user turn:

1. attachment.type = "skill_listing"
   A bullet list of every available Skill with its name + trigger description
   (happy-marriage, gen-task-doc, find-skills, skill-creator, update-config, etc).
   This is how skills get surfaced to the model without bloating the base prompt.

2. attachment.type = "mcp_instructions_delta"
   addedNames + addedBlocks describing connected MCP servers. Observed: a "figma"
   server block with CAPABILITIES / WHEN TO USE sections injected as instructions.

3. attachment.type = "task_reminder"
   {content: [], itemCount: 0} — the live TODO/task list state, injected so the
   model stays aware of outstanding tasks (the "system-reminder" mechanism).

Also present: `ai-title` records (an auto-generated conversation title) and
`last-prompt` / `queue-operation` bookkeeping records.
